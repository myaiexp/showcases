Motion One

motion.dev/docs →

A tiny, performant animation library built on the Web Animations API. Springs, staggers, and timelines in under 3kb.

Animation
Why it's interesting: Motion One wraps the browser's native Web Animations API with a friendly, powerful interface. Animations run off the main thread, so they stay smooth even when JS is busy. It's a fraction of the size of GSAP or Anime.js, supports spring physics out of the box, and has a stagger utility that makes sequenced animations trivial.

Uses animate() with spring easing

View Code 
import { animate } from "motion";

box.addEventListener("click", () => {
  // Scale up and rotate with a spring, then back
  animate(
    box,
    { scale: [1, 1.3, 1], rotate: [0, 15, 0] },
    { duration: 0.6, easing: "ease-out" }
  );
});

Stagger Entrance

Hit the button to reset and replay a staggered entrance animation across a row of elements.

View Code 
import { animate, stagger } from "motion";

animate(
  ".stagger-item",
  { opacity: [0, 1], y: [20, 0] },
  { delay: stagger(0.08), duration: 0.5, easing: "ease-out" }
);

Pointer-follow Card

Drag or move your finger over the area — the card smoothly follows using animate().

Drag

Uses animate() on each pointer event for smooth trailing

View Code 
import { animate } from "motion";

zone.addEventListener("pointermove", (e) => {
  const rect = zone.getBoundingClientRect();
  const x = e.clientX - rect.left - card.offsetWidth / 2;
  const y = e.clientY - rect.top - card.offsetHeight / 2;

  animate(card, { x, y }, { duration: 0.3, easing: "ease-out" });
});
